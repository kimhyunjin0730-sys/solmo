# 챗봇 지식 기반 및 응답 로직 리서치 보고서

## 1. 개요
현재 (주)솔모정보기술 웹사이트의 챗봇(솔모봇)이 "회사 소개" 및 "제품 소개"와 같은 기본적인 질문에 적절히 답변하지 못하는 현상을 분석한다.

## 2. 구성 요소 분석

### A. Frontend (`src/components/ChatBot.js`)
- 유저 입력을 `/api/chat`으로 전송.
- 응답 대기 중 로딩 상태 표시.
- 오류 발생 시 "일시적인 오류가 발생했습니다"라는 하드코딩된 메시지 출력.

### B. Backend API (`src/app/api/chat/route.js`)
- **OpenAI API 연동**: `gpt-4o-mini` 모델 사용.
- **Fallback 메커니즘**: 
    - `OPENAI_API_KEY`가 없거나 API 호출 실패 시 `fallbackReply()` 함수 호출.
    - 현재 `fallbackReply()`에는 "회사 소개"나 "제품 소개"에 대한 키워드 처리가 누락되어 있음.
    - 단순히 "안녕하세요! 솔모정보기술 챗봇입니다. 자세한 상담은..." 이라는 기본 메시지만 반환함.

### C. 지식 베이스 (`src/lib/knowledge.js`)
- `my-js-web` 디렉토리의 모든 `.md` 파일을 읽어 시스템 프롬프트에 주입.
- `STATIC_INFO` 상수에 회사 기본 정보(전화, 주소, 주요 고객사 등)가 정의되어 있음.
- 파일 읽기 실패 시 경고만 출력하고 빈 섹션으로 넘어감.

## 3. 문제 원인 추정
1. **Fallback 의존**: 네트워크 문제나 OpenAI API 할당량/키 문제로 Fallback이 실행될 경우, 기본 정보 응답이 불가능한 구조임.
2. **프롬프트 가이드 부족**: 시스템 프롬프트에서 "회사가 무엇인지", "무슨 제품이 있는지"에 대한 요약 답변을 강조하지 않아 모델이 지식 베이스의 방대한 텍스트 중 핵심을 짚지 못할 가능성.
3. **지식 베이스 누락**: `my-js-web/회사 소개.md`나 `제품소개.md` 파일이 존재하지만, 챗봇이 이를 효과적으로 "답변용"으로 가공해서 읽지 못할 수 있음.

## 4. 해결 방안
- `fallbackReply`에 핵심 회사 정보 및 제품 리스트 추가.
- `buildSystemPrompt` 개선: "회사가 무엇인지", "주요 솔루션이 무엇인지"에 대한 우선순위 가이드 추가.
- `loadKnowledge` 로직 검증: 파일 인코딩이나 경로 문제로 파일을 못 읽고 있는 것은 아닌지 재확인.

## 5. 결론
가장 가능성 높은 원인은 **Fallback 로직의 부실함** 또는 **시스템 프롬프트의 불분명함**이다. 이를 보강하여 지능적인 답변이 가능하도록 수정이 필요하다.
