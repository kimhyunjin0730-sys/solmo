# Development Process Rules

## [개발 원칙]
1. 작성된 계획을 사용자가 직접 검토 하고 승인 하기 전까지 절대 코드 작성을 하지 않는다.

## [개발프로세스]

### 1. 코드리서치
- 발견한 내용을 `Documents/research.md` 파일을 만들어서 모두 100% 검증하고 기록한다. (내용을 대표하는 제목 사용)
- 프로젝트 내용을 깊이 읽고, 동작 방식을 이해하며, 모든 세부사항을 파악한 후 상세 보고서를 반드시 업데이트한다.

### 2. 계획
- 별도의 마크다운 파일에 상세 구현 계획을 작성한다. (Artifact 파일과 함께 md 파일로 생성 후 소통)
- 특정 기능에 대해 상세 플랜이 나오면 `doc/plan.md` 파일을 반드시 기록한다.
- 코드 스니펫, 기능별 명칭 사용 기록 등을 포함한다.
- **오프셋 페이징 대신 인풋 페이징을 지원해야 한다.**
- 실제 코드 베이스를 기반으로 접근 방식, 구조, 파일 경로, 트레이드오프를 상세히 기술한다.

### 3. 주석달기 및 계획 업데이트
- Artifact 업데이트 시 `plan.md` 파일도 동시 업데이트한다.
- 요구사항 추가 시 즉시 기록하고 Antigravity 에이전트에게 전달한다.
- 모든 과정이 끝나기 전에는 절대 구현을 시작하지 않는다.

### 4. 구현
- 모든 작업과 단계를 중단 없이 전부 구현한다.
- 완료된 단계는 계획 문서에 '완료'로 표시한다.
- **`any` 나 `unknown` 타입을 사용하지 않는다.**
- 지속적으로 `typecheck`를 실행하여 새로운 문제를 방지한다.
- 매 단계 종료 시 버전 명을 정하여 Git에 업데이트한다.

### 5. 피드백
- 정확한 테스트를 실제로 구동하고 모든 항목을 검증한 후 피드백을 진행한다.
